📊 Comparación para el sitio

Característica OnlineGDB OneCompiler
Código precargado (ejemplos de teoría) ✓ Sí — guardás el ejemplo, compartís el link ✗ No — siempre arranca en blanco
Alumno puede editar y ejecutar ✓ Sí ✓ Sí
Interfaz completa (guardar, AI, etc.) ~ Solo run y tema ✓ Completa (guardar, compartir, AI)
Permite código inicial en ejercicios ✓ Sí (podés dar código "base") ✗ No — solo vacío
Cómo obtener el embed Share → Embed Code → copiar <script> URL fija: onecompiler.com/embed/java
¿Cambia si editás el programa original? ✓ Sí — el embed muestra siempre tu versión actual —
Uso ideal en el sitio Teoría: mostrar ejemplos listos para ejecutar Ejercicios: alumno escribe la solución

💡 Combinación sugerida: usar OnlineGDB en el tab de Teoría (con ejemplos precargados) y OneCompiler en el tab de Ejercicios (editor libre para que el alumno resuelva). Así las pildoras de código de los ejemplos están siempre listas y los alumnos tienen lienzo en blanco para practicar.
